## 四、各 Step 功能明细

### Step 1 — Input & Parse

- 文本粘贴、文件上传、`加载 AUT 15 条样本`（空 content 触发全量 ingest）
- 调用 `POST /ingest` + 对每条 `POST /parse`
- 行内编辑：`input_fields`、`expected_action`、`conditions`
- 「确认」标记 designer_confirmed
- Prompt 透明度折叠面板（`prompt_template_id`、`source_context_ids` 等，依赖 B 字段）
- 修订记录 + ImprovementSummary

### Step 2 — Risk Analysis

- `POST /risk`：热力图（点击筛选联动明细表）、Score、I×L、Priority 下拉覆盖
- `POST /coverage`：覆盖项表可编辑 description / techniques / strategy_rationale
- 「新增覆盖项」按钮（`COV-AUT-DESIGN-xxx`，记入 revisions）

### Step 3 — Generate & Evaluate

- `POST /generate`：用例表可编辑 title、test_steps、expected_result；Draft/Approved/Rejected
- 按 technique、status 筛选；EP/BVA/DT/FSM 数量标签
- `POST /oracle`：独立 Oracle 表 + needs_review 时「Designer 确认」
- `POST /fsm`：Mermaid 图 + 路径覆盖状态下拉
- TraceabilityPanel：点击 Req ID 展示需求 → 覆盖项 → 用例链
- 生成超过 2s 显示 NFR 提示

### Step 4 — Optimize & Export

- `POST /optimize`（仅 Approved 的 test_ids）
- 「导出预览」Modal + 导出 Approved JSON/CSV
- 导出包含：`test_cases`、`risk_scores`、`coverage_items`、`revisions`（D 契约格式）
- ImprovementSummary + RevisionPanel
