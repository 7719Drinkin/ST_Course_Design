## 二、前端功能（4 步 Pipeline）

### Step 1 — Input & Parse（FR1.0 / FR1.1）
- 文本粘贴 + 文件上传
- 调用 `POST /ingest` + `/parse`
- 行内编辑 `expected_action`、`conditions`
- 「确认」按钮清除 missing_fields 告警
- 修订写入 `revisions`

### Step 2 — Risk Analysis（FR2.0）
- `POST /risk` 驱动 5×5 热力图（可点击筛选）
- Priority 下拉人工覆盖
- `POST /coverage` 展示覆盖项与测试策略（Mainly）

### Step 3 — Generate & Evaluate（FR3 / FR4 / FR5）
- `POST /generate` 用例表：编辑 expected_result、改 risk、Draft/Approved/Rejected
- `POST /oracle` 置信度展示
- `POST /fsm` Mermaid 文本
- 点击 requirement_id 追溯高亮

### Step 4 — Optimize & Export（FR6 / FR7）
- `POST /optimize`（基于 Approved test_ids）
- `POST /export` 仅导出 Approved + revisions
- 展示 `removed_test_ids`

## 四、运行方式

```bash
# 终端 1：后端
pip install -r backend/requirements.txt
uvicorn backend.main:app --reload --port 8000

# 终端 2：前端
cd frontend && npm install && npm run dev
```

访问 http://localhost:5173 ，各 Step 应显示绿色 **Live**。
