# 小组分工（更新版 — 含 Interactive Review 要求）

## Scope（项目范围）

本项目为 **AutoTestDesign**，目标是在 2 周 Sprint 内完成一个面向软件测试设计的智能化系统原型。系统以 AUT SRS（被测系统需求规格说明）和标准测试知识库为输入，结合 RAG、LLM、测试设计算法与前端可视化能力，自动生成、评估、优化并导出测试用例。

**AUT（被测目标应用）：** msaswata15/LibraryManagementSystem，branch: copilot/fix-398405b9-7572-4792-9850-4f677476485c，commit: 2188543。

> **重要说明（更新版作业要求）：** Risk Analysis Report、Test Plan、Detailed Test Design 三份文档的描述对象均为 AUT（LibraryManagementSystem），而非 AutoTestDesign 工具本身。工具架构详情放 README 与演示视频，不放在上述三份文档中。

项目核心功能范围包括：

1. **需求输入与解析（FR 1.0 / FR 1.1）**
   - 支持 CSV / TXT / AUT SRS 文本输入；
   - 将原始需求归一化为内部 JSON；
   - 使用 LLM Prompt 抽取需求字段，并校验字段完整性。

2. **风险分析（FR 2.0）**
   - 基于 Impact × Likelihood 风险矩阵输出 High / Medium / Low 风险等级；
   - 结合 RAG 检索和 LLM 建议生成风险评分；
   - 在前端以热力图方式展示风险分布。

3. **黑盒测试设计生成（FR 3.0）**
   - 支持等价类划分 EP、边界值分析 BVA、决策表 DT；
   - 自动生成测试用例；
   - 为测试用例补充标准引用 standard_ref。

4. **白盒 / 状态机测试设计（FR 4.0）**（可选加分）
   - 从需求中抽取状态四元组；
   - 构建 FSM；
   - 支持 All-States / All-Transitions 覆盖路径；
   - 输出 Mermaid stateDiagram-v2 图。

5. **测试 Oracle 与评估（FR 5.0）**（可选加分）
   - 使用 LLM 语义判断与规则断言结合的双层 Oracle；
   - 输出 confidence 分数；
   - 对低置信度样例进行人工复核标记。

6. **测试用例导出（FR 6.0）**
   - 支持 JSON、CSV、XLSX 导出；
   - 导出字段包括 test_id、technique、standard_ref、risk_level、status、revision_count；
   - Excel 导出需支持表头冻结与风险等级条件格式。

7. **测试套件优化（FR 7.0）**（可选加分）
   - 使用贪心 set-cover 算法进行用例最小化；
   - 支持风险优先排序模式；
   - 前端展示优化前后数量对比。

8. **Interactive Review（强制要求 — 更新版新增）**
   - 工具必须支持以下完整交互流程：
     概念 → 覆盖项识别 → 覆盖策略 → 测试用例 + 可追溯性 → Prompt 设计 → 结果分析 → 基于证据的改进；
   - 测试人员可在线修改覆盖项、策略、测试用例，修改记录需保存；
   - 工具需维护 DesignSession（设计会话）、CoverageItem（覆盖项）、RevisionRecord（修改记录）三个核心数据结构。

9. **文档与交付范围**
   - 完成 Risk Analysis Report、Test Plan、Detailed Test Design 三份主要文档（均针对 AUT）；
   - 完成 README、演示脚本、提交清单与演示视频；
   - 所有成员在共享文档中完成自己负责章节，D 只负责格式一致性和占位符检查，不负责内容合并。
