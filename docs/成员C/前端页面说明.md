# 前端交互与页面说明文档 (Pipeline Architecture)

基于最终确认的“流水线 / 步进式工作流 (Pipeline)”方案，本系统整体交互形态被设计为一个 4 步走的全局向导 (Stepper)。左侧固定为导航及进度指示侧边栏，右侧主视觉区承载各步骤对应的操作。

这种组织方式将 **测试生成**（包含黑盒用例与白盒状态机）和 **Oracle 评估** 整合在同一屏幕内，有效缩短了用户在“生成 - 审查 - 确认”这一连贯动作中的切换路径。

以下为 4 个主要流程步骤的页面内容说明：

---

## 1. Input & Parse (FR1.0, FR1.1)
**主要职责**：需求文件的载入与第一层结构化转换。

### 包含内容
- **上传工作区 (FR1.0)**：
  - 核心组件：支持拖拽的 `Upload.Dragger`。
  - 功能支持：接受 `CSV/TXT` 格式文件，作为 `AUTSRS`（系统需求规格说明）原始输入。
- **解析结果面板 (FR1.1)**：
  - 核心组件：`Table` 需求数据网格 与 `Alert` 错误反馈区。
  - 数据呈现：将 LLM 解析后的内容通过表格呈现（含 Requirement ID, Feature, Actor, Precondition, Acceptance Criteria）。
  - 用户行为：在此处查阅遗漏的非功能约束字段等告警提示，并能快速重置或走向下一步。

---

## 2. Risk Analysis (FR2.0)
**主要职责**：让用户在进行任何用例设计前，从宏观上洞察当前需求池的风险分布。

### 包含内容
- **风险热力图 (Heatmap)**：
  - 基于 `Impact × Likelihood` 生成的 5x5 矩阵视图。
  - 利用颜色分级（绿色 Low、黄色 Medium、红色 High）呈现出各象限包含的需求数量密度。
- **风险总览快照 (Snapshot)**：
  - 用数据卡片展示：高风险需求数、中风险需求数、低风险需求数。
  - 明确标出最高风险象限的具体定位。

---

## 3. Generate & Evaluate (FR3.0, FR4.0, FR5.0)
**主要职责**：【核心工作区】集中展示通过各种算法生成的测试用例，并同步利用 Oracle 模型为其打分，提供给测试工程师复核。

### 包含内容
- **综合用例数据表 (FR3.0 黑盒生成)**：
  - 包含等价类(EP)、边界值(BVA)、决策表(DT)产生的各类测试用例。
  - 具备富交互（筛选、排序、分组），可按 `Risk Level` 或 `Technique` 过滤。
  - 表中直接展示对应的标准引用（Standard Ref）。
- **FSM 状态机可视化面板 (FR4.0 白盒生成)**：
  - 利用 `Mermaid stateDiagram-v2` 渲染系统抽取出的状态机路径。
  - 支持辅助查阅白盒 All-States / All-Transitions 用例背后的业务逻辑链路。
- **Oracle 评估反馈面板 (FR5.0)**：
  - 在当前页呈现每条用例的 `LLM Verdict` 和 `Rule Verdict`。
  - 呈现系统自动计算出的 置信度分数（Confidence Score）。
  - **Review Status**：系统会自动高亮低置信度用例，并抛出“待人工复核”的行动点，用户可在此界面进行一键判定/修改。

---

## 4. Optimize & Export (FR7.0, FR6.0)
**主要职责**：对确认无误的庞大测试套件进行去重优化，并导出为最终交付物。

### 包含内容
- **套件优化控制台 (FR7.0)**：
  - 核心组件：Set-Cover 贪心算法的触发器，以及优化模式切换开关（`风险优先模式` vs `标准模式`）。
  - **Before / After 对比面板**：通过数据大卡片直观对比“优化前”与“优化后”的用例总数。
  - **缩减率指标 (Reduction Rate)**：利用进度条直观展示套件的瘦身比例。
- **测试结果交付区 (FR6.0)**：
  - 在页面顶端/关键位置提供“一键导出”行动点。
  - 将最终确定的用例池打包转换为 JSON / CSV / XLSX，并包含全量属性（test_id, technique, standard_ref, risk_level 等）。
