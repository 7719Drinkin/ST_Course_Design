## 二、前端架构

### 布局方案

采用"流水线 / 侧边 Stepper"方案（方案一），左侧固定 280px Sidebar 内嵌垂直 Steps 导航，右侧为各步骤独立内容区。

```
┌─────────────┬──────────────────────────────────────────┐
│  Sidebar    │  Header                                   │
│             ├──────────────────────────────────────────┤
│  Step 1     │                                           │
│  Step 2  ◄──┤  当前步骤内容区                            │
│  Step 3     │                                           │
│  Step 4     │                                           │
└─────────────┴──────────────────────────────────────────┘
```

### 目录结构

```
frontend/src/
├── App.tsx                  # 主应用，4 个 Step 组件
├── main.tsx                 # 入口，ConfigProvider 主题配置
├── index.css                # 全局样式（CSS 变量 + 布局样式）
├── store/
│   └── appStore.ts          # Zustand 全局状态（currentStep）
├── types/
│   └── index.ts             # 所有共享类型（对齐 integration_interfaces.md）
└── services/
    └── api.ts               # API 服务层（真实调用 + mock fallback）
```

## 四、API 服务层设计

**文件**：`frontend/src/services/api.ts`

- 所有 API 调用通过统一的 `ApiResult<T>` 包装返回
- `isLive: true` = 真实后端数据；`isLive: false` = 本地 mock 兜底
- UI 层根据 `isLive` 显示绿色 "Live" 或橙色 "Mock · 待接入" 标签
- mock 数据字段严格对齐 `docs/integration_interfaces.md` 中的接口规范

| 函数 | 接口 | 等待 |
|------|------|------|
| `ingestAndParse()` | `POST /ingest` + `/parse` | A + B |
| `generateFSM()` | `POST /fsm` | A |
| `getTestCases()` | `POST /generate` | E |
| `getOracleResults()` | `POST /oracle` | B |
| `getRiskData()` | `POST /risk` | B |
| `getOptimizeResult()` | `POST /optimize` | E |
| `getExportUrl()` | `GET /export/{format}` | A |

## 五、类型系统

**文件**：`frontend/src/types/index.ts`

所有类型均与 `docs/integration_interfaces.md` 对齐，供其他成员接口对接时参考：

- `DisplayRequirement` — 需求解析展示结构（Step 1）
- `TestCase` — 测试用例结构（E 输出，Step 3）
- `FSMResult` / `FSMTransition` — 状态机结构（A 输出，Step 3）
- `OracleResult` — Oracle 评估结果（B 输出，Step 3）
- `RiskEntry` — 风险条目（B 输出，Step 2）
- `OptimizeResult` — 优化结果（E 输出，Step 4）
- `ApiResult<T>` — 通用 API 结果包装器

## 七、运行方式

```bash
cd frontend
npm install
npm run dev        # 开发服务器，默认 http://localhost:5173
npm run build      # 生产构建
```

后端地址配置（可选，默认 `http://localhost:8000`）：

```bash
# frontend/.env.local
VITE_API_BASE=http://localhost:8000
```
